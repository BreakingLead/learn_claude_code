# Agent Builder Primary Path

Agent Builder will optimize its default UI for creating, validating, and dry-running a single Agent Blueprint, not for multi-agent workflow orchestration or JSON-first editing. The first screen should use a left asset library, central Blueprint canvas, and right contextual Inspector; the underlying JSON remains available as a bottom drawer for advanced inspection and manual edits. New Blueprints should start with a root Agent node already present so the user can add instructions, capabilities, constraints, and logic around a clear anchor. Multi-agent workflows should move to a separate top-level area because they model dataflow between agents rather than one agent's capabilities. This keeps the product demonstrable in minutes while still preserving access to the declarative format.
